Subsessions and work sessions¶

subsessions.py¶

subsessions.get_subsessions(infile, outfile, threshold=3, cleaned=None)[source]¶

Clusters data into work sessions and subsessions.

Subsessions are separated by launch events. Work sessions are separated by a gap of 3 hours without activity.

Keyword arguments: infile – the input file (CSV) outfile – the resultant file (CSV)

subsessions.to_int(num)[source]¶

Converts a string to an integer; can handle scientific notation.

Will throw an exception if string is not an integer in scientific notation or otherwise.

work_sessions.py¶

Collapses subsessions into work sessions.

To use:
./work_sessions.py <subsessions file> <outfile>
work_sessions_from_subsessions.get_work_sessions(infile, outfile)[source]¶

Collapses subsession data from infile into work session data in outfile.

Gets work session data from raw sensordata. Work sessions are separeted by a specified number of hours of inactivity (3 by default).

To use:
from sensordata import worksessions or ./work_sessions_from_raw.py <infile> <outfile> on the command line
work_sessions_from_raw.userworksessions(usergroup, threshold=3)[source]¶

Acts on sensordata for a single student-project. Returns a dataframe of work sessions, where work sessions are delimited by threshold hours of inactivity.

Keyword arguments: usergroup pd.DataFrame containing all sensordata from the given project threshold the number of hours of inactivity that should separate a work session.

default 3.
work_sessions_from_raw.worksessions(infile, outfile=None)[source]¶

Given raw sensordata for all students, group events for individual student-projects and compute work sessions.

Returns summarised work sessions, which includes start times, end times, the number of edits (test and normal) and the number of launches (test and normal).
