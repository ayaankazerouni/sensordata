Subsessions and work sessions¶

Subsessions¶

subsessions.get_subsessions(infile, outfile, threshold=3, cleaned=None)[source]¶

Clusters data into work sessions and subsessions.

Subsessions are separated by launch events. Work sessions are separated by a gap of 3 hours without activity.

Parameters:
  • infile (str) – path to the input file (CSV)
  • outfile (str) – path to the resultant file (CSV)

Work sessions¶

work_sessions.summarise_worksessions(df)[source]¶

Acts on sensordata for a single student-project.

Returns a dataframe of work sessions, where work sessions are delimited by some hours of inactivity. This is typically called by worksessions as part of a split-apply-combine procedure.

Parameters:df (DataFrame) – containing all sensordata from the given project
Returns:A DataFrame containing the work sessions for the given user on a given assignment.
work_sessions.worksessions(infile, outfile=None)[source]¶

Given raw sensordata for all students, group events for individual student-projects and compute work sessions.

Returns summarised work sessions, which includes start times, end times, the number of edits (test and normal) and the number of launches (test and normal).
